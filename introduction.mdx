Proyecto Colegio is a comprehensive school administration platform that centralizes student enrollment, teacher management, academic planning, and attendance tracking in a single web application. Built with Next.js 15, Material UI, and Prisma on PostgreSQL, it provides school administrators with real-time data management and reporting tools.

## What Proyecto Colegio does

Proyecto Colegio covers the full administrative workflow of a K-12 school:

- **Student registry** — Store student demographics, contact information, and representative details. Assign students to sections as they enroll each school year.
- **Teacher registry** — Record teacher credentials, specialties, hiring dates, and positions. Assign academic loads linking each teacher to specific sections and subjects.
- **Academic structure** — Define grade years (1st Year, 2nd Year, etc.), create sections per year and school period, build subject catalogs, and attach subjects to grade-level study plans.
- **Attendance** — Take daily attendance for students and teachers with status codes (Present, Absent, Justified, Late) and optional per-subject detail. Review and edit records in a monthly calendar view.
- **Reports** — Export attendance reports and student/teacher lists to PDF and Excel directly from the browser.

## Technology overview

| Layer | Technology |
|---|---|
| Frontend | Next.js 15 (App Router), React 19, Material UI 7 |
| Backend | Next.js Server Actions (no separate API layer) |
| Database | PostgreSQL via Prisma ORM 6 |
| Reports | jsPDF, jspdf-autotable, xlsx |
| Styling | MUI theming system |

<Note>
Proyecto Colegio uses Next.js Server Actions exclusively — there are no REST API endpoints or separate backend service. All data mutations happen through typed server-side functions called directly from client components.
</Note>
